HTML5基本结构

1.简洁的文档声明

  • XHTML 严格的文档声明:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

  • HTML5的文档声明:

<!DOCTYPE html>

2.页面语言:

  • 指明网页中使用的自然语言
  • lang=”en” –指定语言为英语
    <html lang="en">

    3、设置页面的字符编码

    charset=”UTF-8” –建议HTML、CSS、JS、后台、数据库等都采用的UTF-8进行字符编码，这样我们前后台之间的交互就不会出现乱码问题，并且在浏览器中展示也不会出现乱码问题(“一键UTF-8到底”规则)

<meta charset="UTF-8" >

4、<title>网站的标题</title>

5、<meta name="keywords" content="网站的关键字内容信息"/>

6、<meta name="description" content="网站的描述内容信息"/>

7、外链式引入我们的样式表文件

<link charset="utf-8" type="text/css" rel="stylesheet" href=""/>

8、做响应式布局

  • 放在</head>前

  • `<meta name=”viewport”

    content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">`
  • 快捷键 meta：vp +tab键

    9、外链式引入我们的JS脚本文件

    <script charset="utf-8" type="text/javascript" src=""></script>

普通的页面布局方式

border-radius 圆角

box-shadow 阴影

border-color 三角形

background - origin

background-origin 属性规定背景图片的定位区域。
背景图片可以放置于 content-box、padding-box 或 border-box 区域。

background-clip

规定背景的绘制区域。

background-size

  • cover:将背景图像等比缩放到完全覆盖容器，背景图像有可能超出容器。

  • contain:将背景图像等比缩放到宽度或高度与容器的宽度或高度相等，背景图像始终被包含在容器内

  • 自定义背景图像大小

    background-size:contain; 按照原始的比例缩放,整个背景图都显示完整 ,按照宽高比较小的去缩放,但是会出现空白

background-size:cover;按照原始比例等比缩放,但是必须铺满整个容器 ,按照宽高比较大的缩放,不会出现空白,但会出现显示不完整

gradient 背景渐变

filter滤镜

css3中的filter除了blur以外还添加了其他的属性

text-shadow 文本阴影

text-overflow 文本溢出

  • 将text-overflow生效必备的3个属性：overflow、width、white-space都直接定义在了内联容器的父级块容器上

  • ellipsis: 将溢出的文字显示省略标记（…）

  • clip: 直接将溢出的文字裁剪

word-wrap 文本换行

  • word-wrap:break-word;
  • word-wrap:normal;

box-sizing

  • box-sizing: border-box; 内容的宽度+padding+border
  • webkit-box-sizing: border-box; 【谷歌写法】
  • box-sizing: content-box; 内容的宽度(box-sizing默认的值)
  • box-sizing: padding-box; 只是火狐浏览器兼容 内容的宽度+padding的值

标准浏览器中:盒子所占据的空间大小 = width+padding+border+margin

怪异的模式(不写文档申明,在ie6-7会变成怪异模式) width包含了(padding和border和内容的宽度)

css3选择器

一、基本选择器

选择器 类型 功能描述
* 通配符选择器 选择文档中所有的HTML元素
Element 元素选择器 选择指定类型的HTML元素
id(#) ID选择器 选择指定ID属性值为”id”的任意类型元素
.class 类选择器 选择指定class属性值为’class’的任意类型的任意多个元素
selector1,selectorN 群组选择器 将每一个选择器匹配的元素集合并

二、层次选择器

选择器 类型 功能描述
E F 后代选择器 选择匹配的F元素,且匹配的F元素被包含在匹配的E元素内
E>F 子选择器 选择匹配的F元素，且匹配的F元素所匹配的E元素的子元素
E+F 相邻兄弟选择器 选择匹配的F元素，且匹配的F元素紧位于匹配的E元素的后面
E~F 通用选择器 选择匹配的F元素，且位于匹配的E元素后的所有匹配的F元素

html5新增标签和兼容处理

在之前的HTML页面中，大家基本上都是用了Div+CSS的布局方式。而搜索引擎去抓取页面的内容的时候，它只能猜测你的某个Div内的内容是文章内容容器，或者是导航模块的容器，或者是作者介绍的容器等等。也就是说整个HTML文档结构定义不清晰，HTML5中为了解决这个问题，专门添加了：页眉、页脚、导航、文章内容等跟结构相关的结构元素标签。

总结：
1、新增加一些构建页面语义化的结构标签 ，对原有 标签进行了修改
header、nav、section、footer、article、aside、figure、figcaption、hgroup…

(在IE6~8浏览器中不识别这些新增加的标签,导致布局结构混乱)在页面中引入

2、给我们的INPUT表单元素增加了很多新的类型(不兼容)

原有:text、password、radio、checkbox、button、submit、reset、file…

新增:search、url、email、tel、number、date、time、color、range…

作用:

1)根据TYPE的类型不一样,会调取出最符合用户输入的物理键盘;

2)传统的表单验证:当某个行为触发的时候,我们获取用户输入的内容和自己编写的正则进行验证从而判断用户输入的是否符合规则;

3)HTML5中提供了CSS3/JS的新验证方式,不需要自己写正则,浏览器自己就已经实现了验证,我们只需要控制成功/不成功的样式或者其他操作即可
